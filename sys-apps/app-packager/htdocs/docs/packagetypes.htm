Package Types

For general use you don't need to think about packaging, the default format is good for most needs. However if needed you can add a "package" property to your .desk file and specify a specialist package format.

Fast Package

 "package":"fast",
By specifying the fast package format the raw contents of each file to be packaged will be dumped into the package one after the other. This offers a very slight performance improvement, it also decreases the size of the package file by up to 40%. However the file may be damaged when transferring via email / ftp / ssh if the transfer program does not transfer the file exactly bit for bit.

Scrambled Package

 "package":"scramble",
Telling deskshell to scramble the package will cause it to encrypt the contents of the file making it unreadable to none-technical users. Study of the deskshell source code will reveal the key used to encrypt the file making it relatively easy to decrypt by technical users. This format is meant as a demonstration of how the package format could be encrypted. A closed source fork of deskshell could hide the key used to encrypt / decrypt the package contents and so make it more secure.

Standard Package

 "package":"standard",
Just for completeness you can also set the package format to standard. If the package property is missing from the .desk file then the package will be the standard format. This format is safe for transfer via ftp / email / ssh.
